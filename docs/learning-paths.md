# Recommended Learning Paths

This document expands on the recommended learning paths outlined in the main README. These paths are designed to give you a structured sequence of technologies to learn for various roles.

## Why follow a path?

Learning technologies in isolation often leads to confusion. For example, learning React before understanding basic JavaScript DOM manipulation will make you a weaker frontend developer. These paths ensure you learn the *prerequisites* before the *frameworks*.

## The Paths

### 1. Complete Beginner
This is for someone who has never written a line of code.
- **Computer Science Fundamentals:** Understand how computers and memory work. (Recommended: Harvard CS50)
- **Git & GitHub:** Learn how to save your code and collaborate.
- **HTML, CSS, JavaScript:** The absolute foundation of the web.
- **Frontend Basics:** Build simple, interactive websites.
- **Backend Basics:** Learn how a server responds to a browser.
- **Databases:** Learn where data is stored (SQL).

### 2. Frontend Developer
- **HTML, CSS, JavaScript:** Master the core trio.
- **Framework (React / Vue / Angular):** Pick *one* and master it. React is currently the most employable.
- **State Management:** Learn how to manage complex data in your app (Redux, Zustand, Pinia).
- **Build Tools & Bundlers:** Understand Vite, Webpack, and NPM.
- **CSS Frameworks:** Tailwind CSS is highly recommended.
- **Testing:** Learn Jest and React Testing Library.

### 3. Backend Developer
- **Programming Language:** Choose Python, Go, Java, C#, or Node.js.
- **Relational Databases:** Master PostgreSQL or MySQL.
- **APIs:** Learn how to build RESTful APIs and understand GraphQL.
- **Caching:** Learn Redis to make your APIs fast.
- **Message Brokers:** Understand asynchronous communication (Kafka, RabbitMQ).
- **Containerization:** Learn Docker to package your apps.

*(For Mobile, DevOps, AI, and Cybersecurity, please refer to the specific roadmaps in the main README.)*
